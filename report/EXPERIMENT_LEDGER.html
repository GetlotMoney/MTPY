实验 #23 — AG-JEPA: masked semantic patch prediction + prior correction 已验证

框架在当前 v5+FAE / K=32 路线旁路接入 AG-JEPA 辅助损失, 不替换主分类路径。
核心逻辑 用真实类别文本在 576 个 CLIP patch 中选 top-k 语义相关 patch, 将其视作被遮区域 target; 用剩余 patch 上下文 + 类文本在 tf_common_dim 抽象空间中预测 target。
反事实项 负类文本走同一 predictor, 通过 margin 约束避免不相关文本也能重构目标 patch: lambda_jepa_neg=0.02, jepa_neg_margin=0.2。
后验先验校准 prior_correction=balanced: 评估时统计全体 cached test logits 的 seen/unseen 概率质量, 估计一个 unseen logit shift。该项属于 test-time / transductive calibration, 论文中必须单独标注和消融。
代码状态 已改: model/MyModel.py (~822 predictor, ~1082 masked, ~1569 loss), tools/helper_func.py (~129), train_VGSR_CUB.py, config/VGSR_cub_gzsl.yaml。已通过 py_compile 和 CPU fake-batch smoke test, 已完成正式实验。
消融结果 (干净 60ep schedule, 控变量) A: AG-JEPA off + PriorCorr off (P3.11) → H=72.45
B: JEPA-only (AG-JEPA on + PriorCorr off, P3.10) → H=72.91 ⭐ → +0.46 H 真实贡献
C: prior-only — 跳过 (D 已证 balanced hurts)
D: JEPA + prior full (AG-JEPA on + PriorCorr balanced, P3.1) → H=72.70 → balanced hurts -0.21
多 seed 锁定 (B 配置): seed=5 (P3.10) 72.91 / seed=42 (P3.12) 72.53 / seed=2024 (P3.13) 72.50 → 3-seed avg=72.65±0.19 ⭐ 新 main baseline
结论 AG-JEPA 训练 loss 保留作主创新 (+0.46 H, 控变量验证); PriorCorrection balanced 在 CUB 不适用 (-0.21 H, negative result, 论文需单独标注)。 yaml 默认: use_ag_jepa: True, lambda_jepa: 0.05, lambda_jepa_neg: 0.02, prior_correction: none。
