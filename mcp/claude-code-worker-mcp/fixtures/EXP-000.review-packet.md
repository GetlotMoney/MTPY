# EXP-000 Claude CLI MCP 冒烟审查包

## 实验元信息

- 实验编号：`EXP-000`
- 实验名称：Claude CLI MCP 默认模式冒烟测试
- 审查目标：验证 MCP 可以在默认 `cli` 模式下启动本机 Claude CLI，并解析固定三轮审查结果。
- 当前范围：只测试 MCP 工具链，不修改 DVSR 模型核心代码，不跑训练。

## 本次改动范围

- `src/index.ts`
  - 默认 worker 模式设为 `cli`。
  - `cli` 模式调用本机 `claude -p`，使用本机 OAuth 登录状态，不要求 API key。
  - `cli` 模式在传入的 `cwd` 内运行，并通过 `--add-dir <cwd>` 授权读取同一项目目录。
  - 默认只开放 `Read` 工具，禁止 `Bash`、`Edit`、`Write`、`MultiEdit`、`NotebookEdit`。
  - 审查包通过 stdin 输入，避免同时传 prompt 参数和 stdin。
  - 机器校验 `Pass 1/2/3 Decision` 与 `Overall Decision` 是否同时存在且一致。
  - 旧兼容格式 `Decision: ACCEPTED|REJECTED` 只在没有三轮字段时使用。
  - `start` 会立即返回 job_id，Claude CLI 在后台执行，Codex 通过 `get` / `wait` 追踪。
  - 失败 fallback Markdown 也输出固定三轮 `REJECTED` 格式。
  - `git status` 使用当前 `cwd` 范围内的 `git status --short -- .`，避免 MCP 子目录 smoke 混入父项目无关改动。
- `scripts/smoke.mjs`
  - dry-run 验证默认模式为 `cli`。
  - 手动队列作为 fallback 显式传入 `mode: "manual"`。
  - `--cli-e2e` / `npm run smoke:cli` 使用 `fixtures/EXP-CLI-E2E.review-packet.md` 真实启动 Claude CLI，轮询任务结果。
  - 快速 smoke 已加入硬断言：工具列表、默认模式、Claude CLI 可用、默认/CLI 不需要 API key、external 需要 API key、dry-run 默认 CLI、manual fallback 解析 ACCEPTED。
- `fixtures/EXP-CLI-E2E.review-packet.md`
  - 专门用于 CLI 协议端到端测试，只验证 stdin、后台执行、stream-json 提取、output 写入和三轮解析，不审查 MCP 源码本身，避免自举循环。
- `package.json`
  - 增加 `smoke:cli` 脚本，用于真实 CLI 端到端验证。
- `README.md`
  - 说明默认 CLI 模式、manual fallback、external API 模式和安全默认值。

## 审查要求

请做固定三轮审查：

1. Pass 1：代码与配置正确性。检查 CLI 参数、stdin 输入、决策解析、权限限制、cwd 行为是否合理。
2. Pass 2：实验工作流可复现性。检查 smoke 包是否足以证明默认 CLI 模式可用，是否避免依赖桌面端会话和 API key。
3. Pass 3：运行计划与安全性。检查是否会误改代码、误运行训练、误泄露 key，是否适合进入后续正式实验。

请不要编辑文件，不要运行训练。

## 期望输出格式

Markdown 结果靠前位置必须包含：

```text
Pass 1 Decision: ACCEPTED 或 REJECTED
Pass 2 Decision: ACCEPTED 或 REJECTED
Pass 3 Decision: ACCEPTED 或 REJECTED
Overall Decision: ACCEPTED 或 REJECTED
```

只有三轮全部 ACCEPTED 时，`Overall Decision` 才能是 `ACCEPTED`。
